# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

- リモートリポジトリは、インターネット上（GitHubが広く普及）で、複数人で開発した内容を共有しながら作業できるリポジトリ
- ローカルリポジトリは開発者のパソコン本体に保存されるリポジトリ

## プッシュとマージの違いは何でしょうか？

- pushはローカルリポジトリの内容をリモートリポジトリに送信する
- mergeは他のブランチの変更内容を、統合する

## コミットとプッシュの違い

- commitはローカルリポジトリに変更の内容を保存する
- pushはローカルリポジトリの内容をリモートリポジトリに送る

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

- 簡潔に命令形の動詞で伝える

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

- ローカルでのマージは、自分で書いたコードをそのまま反映させるので、個人的な活動で使う。
- ローカルでの変更内容をリモートリポジトリに反映させる前に、レビューを依頼する。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

- コンフリクトを起こしている相手のコードを把握し、意図が読み取れない場合は、コミュニケーションを取り解消する
